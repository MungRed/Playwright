## 功能说明

自动分析当前项目结构与代码，将 README.md 同步更新为与实际项目一致的文档。

## 执行步骤

1. **收集项目信息**：
   - 读取当前 `README.md` 内容，了解已有文档结构
   - 列出项目根目录结构（`list_dir`）
   - 阅读核心源代码文件，理解实际功能与模块划分
   - 读取示例脚本或配置文件，确认格式规范是否有变化

2. **在修改前，询问用户以下问题：**
   - 📋 **更新范围**：更新全部章节，还是只更新指定章节（如功能特性、项目结构、脚本格式等）？
   - 🌐 **语言**：使用中文还是英文（或保持现有语言）？
   - ➕ **补充信息**：是否有需要额外补充说明的内容？

3. **对比分析差异**，找出 README 与实际代码不一致的地方：
   - 功能列表是否完整
   - 项目结构是否与实际目录匹配
   - API / 脚本格式 / 字段说明是否准确
   - 操作说明（快捷键等）是否正确
   - 示例是否仍然有效

4. **更新 README.md**：
   - 保持原有文档风格和结构
   - 只修改有差异或需要补充的部分，不破坏已有格式
   - 新增章节放在合适的位置

5. **输出更新摘要**：说明修改了哪些章节、更新了哪些内容。

## 注意事项

- 优先以代码为准，README 应反映代码的真实状态
- 保持文档简洁，避免过度描述实现细节
- 如有不确定的内容（如版本、依赖），询问用户而非猜测
- 不要删除用户手动维护的章节（如 License、致谢等），除非用户明确要求
